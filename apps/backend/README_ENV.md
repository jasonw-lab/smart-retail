# .envファイルの使用方法

## ローカル開発環境での.envファイルの使用

このプロジェクトでは、ローカル開発時に環境変数を設定するために`.env`ファイルを使用できます。

### 設定方法

1. プロジェクトのルートディレクトリ（`pom.xml`と同じレベル）に`.env`ファイルを配置します。
   - すでに`.env`ファイルが作成されています。必要に応じて値を変更してください。

2. `.env`ファイルには、アプリケーションが使用する環境変数が含まれています：
   ```
   # Redisパスワード
   REDIS_PASSWORD=123456

   # データベース接続情報
   MYSQL_ROOT_PASSWORD=123456

   # MinIO認証情報
   MINIO_ROOT_USER=minioadmin
   MINIO_ROOT_PASSWORD=minioadmin
   ```

3. アプリケーションを起動する際に、これらの環境変数が自動的に読み込まれます。

### 注意事項

- `.env`ファイルはバージョン管理システム（Git）に含めないでください。
- 本番環境では、`.env`ファイルではなく、システム環境変数を使用することをお勧めします。

## Docker環境での環境変数の使用

Dockerを使用する場合は、`docker`ディレクトリにある`.env`ファイルを使用します。詳細については、`docker/README.md`を参照してください。
