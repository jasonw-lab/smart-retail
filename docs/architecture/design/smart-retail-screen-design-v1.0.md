# Smart Retail 画面設計書 v1.0

> **対象範囲**: Phase 1（コア機能）の画面設計
> **対象ロール**: 本部運営管理者、店舗オペレーター

---

## 1. 画面設計の基本方針

### 1.1 デザイン原則

| 原則 | 説明 | 適用例 |
|------|------|--------|
| **一目で状況把握** | 重要情報は画面上部に集約 | KPIカードをダッシュボード最上部に配置 |
| **異常の即座認識** | 色とアイコンで視覚的に警告 | アラートに優先度別カラーコード |
| **3クリック以内で到達** | 主要機能へのアクセスを最短化 | グローバルナビゲーション + ドリルダウン |
| **現場目線の用語** | IT用語を排除し業務用語を使用 | 「在庫切れ」「賞味期限」「補充」 |

### 1.2 カラーパレット

| 用途 | 色 | カラーコード | 使用場面 |
|------|-----|-------------|---------|
| 正常 | 🟢 緑 | `#67C23A` | 正常稼働、在庫充足 |
| 注意 | 🟡 黄 | `#E6A23C` | 注意喚起、P3アラート |
| 警告 | 🟠 橙 | `#F56C6C` | 要対応、P2アラート |
| 緊急 | 🔴 赤 | `#F56C6C` | 緊急対応、P1アラート |
| 情報 | ⚪ 灰 | `#909399` | 参考情報、P4アラート |
| 主要操作 | 🔵 青 | `#409EFF` | 主要ボタン、リンク |

### 1.3 レスポンシブ対応

| デバイス | 画面幅 | レイアウト |
|----------|--------|-----------|
| デスクトップ | 1200px以上 | 4カラムグリッド |
| タブレット | 768px〜1199px | 2カラムグリッド |
| モバイル | 767px以下 | 1カラムスタック |

### 3.3 UI コンポーネント詳細

|------|---------|----------------|

| 項目 | 表示内容 | インタラクション |

* **店舗×商品別の在庫状況**を確認
* **在庫切れ・期限接近**商品を素早く特定

### 6.2 画面レイアウト（アコーディオンUI）

> **表示方式**: SKU単位での集約表示をデフォルトとし、行クリックでロット別明細を展開（ドリルダウン）
> - 集約行: 合計在庫数 + **最古の賞味期限（在庫が残っているロットのMIN）** を表示

> **ロット（賞味期限）管理方針**:
> - 同一商品（同一SKU）でも賞味期限が異なるため、在庫は **ロット単位**で管理する
> - 画面のデフォルト表示は「パッと見」を優先しSKU集約、詳細が必要な場合にロット別へ展開する

### 6.3 UI コンポーネント詳細

#### 6.3.1 フィルタ条件

| 項目 | タイプ | 選択肢 |
|------|--------|--------|
| 店舗 | ドロップダウン | すべて / 店舗名リスト |
| 商品名 | テキスト入力 | 部分一致 |
| 状態 | ドロップダウン | すべて/在庫切れ/期限接近/在庫過多/正常 |

#### 6.3.2 状態表示ルール

> **注意**: この「状態」は**在庫の状態**を示すものであり、**アラート優先度（P1-P4）**とは異なります。
> - アラート優先度は「対応の緊急度」を示し、主力商品か否か等で判定（要件定義書 5.2.1 参照）
> - 在庫状態は「在庫量・期限の状況」を視覚的に示すもの

**状態判定の優先順位（Phase 1）**:
複数条件が該当する場合は上位を優先する。

| 優先度 | 状態 | 条件 | 表示色 | 備考 |
|--------|------|------|--------|------|
| 1 | 期限切れ | `expiry_date < today` のロットあり（`quantity > 0`） | 🔴 赤 | Phase 1：表示のみ、Phase 2：EXPIREDアラート生成 |
| 2 | 在庫切れ | 合計在庫数 ≤ 発注点 | 🔴 赤 | アラート優先度は商品重要度で P1/P2 に分かれる |
| 3 | 期限接近 | 最古期限 ≤ 3日後 | 🟠 橙 | アラート優先度は残日数で P1/P2/P3 に分かれる |
| 4 | 在庫過多 | 合計在庫数 ≥ 適正上限×1.5 | 🟡 黄 | アラート優先度は P3 固定 |
| 5 | 正常 | 上記以外 | 🟢 緑 | アラート対象外 |
