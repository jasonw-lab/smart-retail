# Smart Retail 画面設計書 v1.3

> **対象範囲**: Phase 1（コア機能）の画面設計
> **対象ロール**: 本部運営管理者、店舗オペレーター

---

## 1. 画面設計の基本方針

### 1.1 デザイン原則

| 原則 | 説明 | 適用例 |
|------|------|--------|
| **一目で状況把握** | 重要情報は画面上部に集約 | KPIカードをダッシュボード最上部に配置 |
| **異常の即座認識** | 色とアイコンで視覚的に警告 | アラートに優先度別カラーコード |
| **3クリック以内で到達** | 主要機能へのアクセスを最短化 | グローバルナビゲーション + ドリルダウン |
| **現場目線の用語** | IT用語を排除し業務用語を使用 | 「在庫切れ」「賞味期限」「補充」 |

### 1.2 カラーパレット

| 用途 | 色 | カラーコード | 使用場面 |
|------|-----|-------------|---------|
| 正常 | 🟢 緑 | `#67C23A` | 正常稼働、在庫充足 |
| 注意 | 🟡 黄 | `#E6A23C` | 注意喚起、P3アラート |
| 警告 | 🟠 橙 | `#F56C6C` | 要対応、P2アラート |
| 緊急 | 🔴 赤 | `#F56C6C` | 緊急対応、P1アラート |
| 情報 | ⚪ 灰 | `#909399` | 参考情報、P4アラート |
| 主要操作 | 🔵 青 | `#409EFF` | 主要ボタン、リンク |

### 1.3 レスポンシブ対応

| デバイス | 画面幅 | レイアウト |
|----------|--------|-----------|
| デスクトップ | 1200px以上 | 4カラムグリッド |
| タブレット | 768px〜1199px | 2カラムグリッド |
| モバイル | 767px以下 | 1カラムスタック |

### 3.3 UI コンポーネント詳細

#### 3.3.1 KPI サマリカード

| 項目 | 表示内容 | インタラクション |
|------|---------|----------------|
| 本日売上 | 金額 + 前日比% | クリックで売上詳細へ |
| 店舗数 | 稼働中/総数 | クリックで店舗一覧へ |
| アラート数 | 未対応件数 | クリックでアラート一覧へ |
| 在庫切れ | SKU数 | クリックで在庫一覧（フィルタ済）へ |

**デザイン仕様**:
- カードサイズ: 幅220px × 高さ120px
- 数値フォント: 28px Bold
- サブテキスト: 14px Regular
- 前日比が正の場合は緑、負の場合は赤で表示
- ホバー時にシャドウを強調

#### 3.3.2 アラートタイムライン

| 項目 | 表示内容 | インタラクション |
|------|---------|----------------|
| 優先度アイコン | P1〜P4の色分けアイコン | - |
| アラート種別 | 在庫切れ/期限接近/在庫過多 | - |
| 店舗名 | 対象店舗 | クリックで店舗詳細へ |
| 時刻 | 検知時刻 | - |
| 商品名 | 対象商品（ある場合）| クリックで商品詳細へ |

**デザイン仕様**:
- 最新10件を表示
- スクロール可能（max-height: 400px）
- 左端に優先度カラーバー（4px幅）
- 未読アラートは背景を薄いハイライト

#### 3.3.3 売上推移グラフ

| 項目 | 仕様 |
|------|------|
| グラフタイプ | 折れ線グラフ（エリア塗りつぶし） |
| 期間切替 | 7日 / 30日 / カスタム |
| 軸 | X軸: 日付, Y軸: 売上金額 |
| ツールチップ | ホバーで日付・売上額・前日比を表示 |

#### 3.3.4 店舗別売上グラフ

| 項目 | 仕様 |
|------|------|
| グラフタイプ | 横棒グラフ |
| 表示内容 | 売上（青）、目標（灰色破線）|
| ソート | 売上降順 |
| 表示件数 | 上位10店舗 |

## 4.5. DV-01 デバイス一覧

### 4.5.1 画面目的

* **店舗に設置されたデバイス（機器）**の登録・管理
* **デバイスの稼働状態・最終通信時刻**を確認
* **デバイス障害の早期検知**を支援（Phase 2 で完全自動化）

> **Phase 1 での扱い**:
> - デバイスの初期登録・編集・削除を行う管理画面として実装
> - `status` と `last_heartbeat` の閲覧・手動更新が可能（Phase 1）
> - Heartbeat 自動監視による `COMMUNICATION_DOWN` / `PAYMENT_TERMINAL_DOWN` アラート生成は Phase 2 で実装（要件定義§8.3.1）

> - 在庫状態は「在庫量・期限の状況」を視覚的に示すもの

**状態判定の優先順位（Phase 1）**:
複数条件が該当する場合は上位を優先する。

| 優先度 | 状態 | 条件 | 表示色 | 備考 |
|--------|------|------|--------|------|
| 1 | 期限切れ | `expiry_date < today` のロットあり（`quantity > 0`） | 🔴 赤 | Phase 1：表示 + 手動廃棄（在庫調整）でゼロ化、Phase 2：EXPIREDアラート生成 |

## 7. A-01 アラート一覧

### 7.1 画面目的

* **全アラートをテーブル形式で一覧**表示し、素早くトリアージ（優先度判断→対応判断）する
* **優先度・状態・種別・店舗でフィルタ・ソート**して対応すべきアラートを特定
* **詳細パネル**でタイムライン・対応メモ・関連画面リンクを確認し、**状態遷移操作を実行**する
* **一括操作**（確認ダイアログ付き）で複数アラートの一括確認・一括対応開始を効率化

> **v1.2 変更**: カード型レイアウトからテーブル型レイアウトに変更。
> 業務管理画面としての情報密度・スキャン性・ソート操作・一括操作性を向上させ、
> 他画面（S-01 店舗一覧、P-01 商品一覧、I-01 在庫一覧）との一貫性を確保する。
>
> **v1.3 変更**: テーブル行内の操作ボタン（確認・対応開始・完了等）を廃止。
> 個別の状態遷移操作は**詳細パネル内でのみ**実行可能とし、誤操作リスクを排除。
> 一括操作は確認ダイアログを追加した上で残す。

### 7.2 画面レイアウト（テーブル型）

> **テーブル型を採用した理由**:
> - アラート一覧は「運用管理画面」であり、大量データの横断比較・ソート・一括操作が必要
> - 要件定義 §4 より「1人あたり5-10店舗を担当」であり、複数店舗のアラートを横断比較する場面が多い
> - カード型は1画面に3〜4件しか表示できず情報密度が低い。テーブル型は10〜20件を一目で把握可能
> - 他画面（S-01, P-01, I-01）もテーブル型であり、UI一貫性を確保する

> - アラートの状態遷移は**不可逆操作**であり、スクロール中やフィルタ操作時の誤タップ/誤クリックの影響が大きい

> - 行クリックでサイドパネルが開く設計のため、追加の操作コストは1クリックのみ
> - テーブルから操作列を除去することで、情報表示領域が広がりスキャン性が向上する

### 7.3 UI コンポーネント詳細

#### 7.3.1 フィルタ条件

| 項目 | 選択肢 | デフォルト値 |

| 優先度 | すべて / P1 / P2 / P3 / P4 | すべて |
| 状態 | 未対応（NEW+ACK+IN_PROGRESS）/ 完了（RESOLVED+CLOSED）/ すべて | **未対応** |

> 画面を開いた瞬間に「今対応が必要なアラート」だけが表示されるべきである。

**デフォルトソート**: 優先度降順（P1が最上位）→ 検知日時昇順（古いものが先）

#### 7.3.2 サマリバー

> フィルタ条件に合致するアラート件数を優先度別に集計表示する。

| 要素 | 説明 |
|------|------|
| P1〜P4 件数 | 各優先度の該当件数を色付きバッジで表示 |
| 合計 | フィルタ結果の総件数 |
| クリック | 該当優先度をトグルフィルタ（クリックでその優先度のみに絞り込み、再度クリックで解除）|

**デザイン仕様**:
- 高さ: 48px
- 背景: `#FAFAFA`（薄灰）
- 件数バッジはそれぞれ優先度カラーの背景 + 白文字
- フィルタ適用中のバッジはアクティブ状態（枠線強調）

#### 7.3.3 アラートテーブル

| カラム | 幅 | 説明 | ソート |
|--------|-----|------|--------|
| ☐ (チェック) | 40px | 一括操作用チェックボックス（ヘッダチェックで全選択）| - |

> **操作列は配置しない（v1.3）**: 個別の状態遷移操作は詳細パネル（7.3.8）内でのみ実行可能。

**行デザイン仕様**:
- 行の左端に **4px幅の優先度カラーバー**（P1:赤, P2:橙, P3:黄, P4:灰）
- 概要列は **2行レイアウト**: 1行目=商品名（14px Bold）、2行目=数値情報（12px Regular, `#909399`）
- 行ホバー時: 背景色を `#F5F7FA` に変更 + **カーソルをpointerに変更**（クリック可能であることを示す）
- 行クリック: 右サイドパネルにアラート詳細を表示（7.3.8 参照）。**これが個別アラートの唯一の操作導線**
- 未確認（NEW）のアラート行: 背景を薄いハイライト `#FEF0F0`

#### 7.3.4 状態バッジ定義

| 内部状態 | 表示ラベル | バッジ色 | 説明 |
|---------|----------|---------|------|
| NEW | 未確認 | 🔴 赤背景（`#F56C6C`）白文字 | 最も目立つ。検知後未確認 |
| ACK | 確認済 | 🟡 黄背景（`#E6A23C`）白文字 | 確認済みだが未着手 |
| IN_PROGRESS | 対応中 | 🔵 青背景（`#409EFF`）白文字 | 作業中 |
| RESOLVED | 完了 | 🟢 緑背景（`#67C23A`）白文字 | 対応済み（検証待ち）|
| CLOSED | クローズ | ⚪ 灰背景（`#909399`）白文字 | アーカイブ済み |

#### 7.3.5 状態別アクションボタン（詳細パネル内）

> **v1.3 設計方針**: 個別の状態遷移操作は**詳細パネル（7.3.8）内でのみ**実行可能とする。
> テーブル一覧には操作ボタンを配置しない（誤操作防止のため）。
> 各状態で「次にやるべきこと」のプライマリアクションは **1つだけ** 強調する。

| 現在状態 | パネル内プライマリボタン | パネル内セカンダリ操作 |
|---------|----------------------|---------------------|
| NEW | [✓ 確認する]（blue, solid）| [▶ 対応開始]（text link） |
| ACK | [▶ 対応開始]（blue, solid）| - |
| IN_PROGRESS | [✅ 完了する]（green, solid）→ 対応完了ダイアログ（7.3.9）| - |
| RESOLVED | [🔒 クローズ]（gray, outlined）| - |
| CLOSED | （ボタンなし）| - |

**ボタンカラーのセマンティクス**:
- 🔵 Blue（solid）= 確認・進行系アクション
- 🟢 Green（solid）= 完了系アクション
- ⚪ Gray（outlined）= クローズ・アーカイブ系

**誤操作防止の補足**:
- 「完了する」ボタンは対応完了ダイアログ（対応内容の入力必須）を経由するため、二重の安全策
- 「クローズ」ボタンは確認ダイアログ（「このアラートをクローズしますか？」）を表示

#### 7.3.6 一括操作（確認ダイアログ付き）

> **設計意図**: 要件定義 §4 より「1人あたり5-10店舗を担当、1日1-2回巡回」。
> 朝一番に複数アラートを一括確認・一括対応開始する運用を効率化する。
>
> **v1.3 変更**: 一括操作は意図的に複数選択→実行する操作なので残すが、
> **確認ダイアログを必須**とし、意図しない一括遷移を防止する。

| 要素 | 説明 |
|------|------|
| チェックボックス | 各行の左端。ヘッダのチェックで表示ページ全件を選択 |
| 一括操作バー | 1件以上選択時にテーブル下部にフローティング表示 |
| 一括確認 | 選択したNEWアラートをACKに遷移（**確認ダイアログ表示**） |
| 一括対応開始 | 選択したNEW/ACKアラートをIN_PROGRESSに遷移（**確認ダイアログ表示**） |
| 選択解除 | 全チェックを解除 |

**確認ダイアログ（一括操作時）**:

```
┌───────────────────────────────────────────┐
│ ⚠️ 一括確認                         [×]  │
├───────────────────────────────────────────┤
│                                           │
│  以下の 3件 のアラートを「確認済」に       │
│  変更してもよろしいですか？                │
│                                           │
│  • P1 在庫切れ - 東京本店                 │
│  • P2 期限接近 - 横浜駅前店               │
│  • P3 在庫過多 - 名古屋栄店               │
│                                           │
│  ※ 対象外のアラート（1件）はスキップ      │
│    されます                               │
│                                           │
├───────────────────────────────────────────┤
│            [キャンセル]  [確認する]       │
└───────────────────────────────────────────┘
```

**バリデーション**:
- 一括確認: NEW以外のアラートが含まれる場合、対象外アラートをスキップし確認ダイアログ内で通知する
- 一括対応開始: CLOSED/RESOLVEDが含まれる場合、対象外アラートをスキップし確認ダイアログ内で通知する

#### 7.3.7 検知日時の相対表示

| 経過時間 | 表示形式 | 例 |
|---------|---------|----|
| 60分未満 | N分前 | 「3分前」 |
| 本日中 | N時間前 | 「2時間前」 |
| 前日 | 昨日 HH:MM | 「昨日 11:00」 |
| 2日以上前 | MM/DD HH:MM | 「01/26 09:00」 |

- ホバーで**絶対日時をツールチップ表示**（例: `2026-01-28 10:30:00`）

#### 7.3.8 アラート詳細パネル（サイドパネル）

> テーブル行クリックで、画面右側にサイドパネルとして表示する。
> テーブルと同一画面内に表示し、画面遷移を発生させない。
> **個別アラートの状態遷移操作はこのパネル内でのみ実行可能**（v1.3）。

```
┌──────────────────────────────────┬────────────────────────────┐
│  アラートテーブル（左側）         │  📋 アラート詳細（右パネル） │
│                                  │                            │
│  [テーブル幅が縮小される]         │  🔴 P1 | 在庫切れ | 対応中  │

│                                  │  ■ 基本情報                │
│                                  │  店舗:    東京本店          │
│                                  │  商品:    プレミアムコーヒー │
│                                  │  ロット:  LOT-2026-0125    │
│                                  │                            │
│                                  │  ■ 数値                   │
│                                  │  現在値:  2個              │
│                                  │  しきい値: 10個（発注点）    │
│                                  │                            │
│                                  │  ■ 状態遷移タイムライン     │
│                                  │  ● 検知    01/28 10:30     │
│                                  │  ● 確認    01/28 11:00     │
│                                  │  ● 対応開始 01/28 11:30    │
│                                  │  ○ 完了    （未）          │

│                                  │  ■ 対応メモ               │

│                                  │                            │
│                                  │  ■ 推奨対応               │

│                                  │                            │
```

**サイドパネル仕様**:
- 幅: 400px（固定）
- アニメーション: 右からスライドイン
- 閉じる: [× 閉じる] ボタンまたはパネル外クリックまたは Esc キー
- テーブル側: パネル表示中はテーブル幅が縮小。選択行はハイライト背景

**パネル表示項目**:

| セクション | 内容 |
|-----------|------|
| 基本情報 | 店舗名、商品名、ロット番号（期限接近の場合）、アラート種別 |
| 数値 | 現在値、しきい値（発注点 or 適正上限）、超過率（在庫過多の場合） |
| 状態遷移タイムライン | 検知→確認→対応開始→完了→クローズの各日時。未到達ステップは○（未完了）で表示 |
| 対応メモ | 完了時に入力された `resolution_note`。未完了の場合は「まだ記録なし」 |
| 推奨対応 | アラート種別に応じた推奨アクション（要件定義書 5.2.1〜5.2.3 の推奨対応） |
| アクションボタン | 現在状態に応じたプライマリアクション（7.3.5 に準拠）。**状態遷移はこのパネル内でのみ操作可能** |
| 関連画面リンク | 在庫一覧（店舗×商品でフィルタ済み）へのリンク |

### 7.4 操作フロー

```
[アラート一覧表示（デフォルト: 未対応フィルタ）]
      │
      ├──[フィルタ・ソート]──▶ テーブルを絞り込み・並び替え
      │
      ├──[サマリバーの優先度クリック]──▶ 該当優先度にトグルフィルタ
      │
      ├──[行クリック]──▶ サイドパネルでアラート詳細を表示
      │    │
      │    ├──[パネル内ボタン]──▶ 状態遷移（確認/対応開始/完了/クローズ）
      │    │                       ※ 完了・クローズは確認ダイアログを経由
      │    │
      │    └──[在庫一覧リンク]──▶ 在庫一覧（店舗×商品フィルタ済み）へ遷移
      │
      └──[チェックボックス選択]──▶ 一括操作バー表示
           │
           └──[一括確認/一括対応開始]──▶ 確認ダイアログ → 実行
```

> **操作導線の設計意図（v1.3）**:
> - テーブル一覧は「閲覧・フィルタ・ソート・選択」に特化（Read-Only）
> - 個別の状態遷移は「行クリック → サイドパネル → ボタン」の3ステップで実行（Deliberate Action）
> - 一括操作は「チェック → ボタン → 確認ダイアログ」の3ステップで実行（誤操作防止）
